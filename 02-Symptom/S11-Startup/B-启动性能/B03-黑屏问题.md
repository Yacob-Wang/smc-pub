# 1. 背景：为什么"黑/白/闪屏"是稳定性头号问题

## 1.1 一句话定位

**启动期黑/白/闪屏 = 用户视觉感知的"启动失败"**——黑屏是 WMS/SF 卡死、白屏是渲染问题、闪屏是过渡不连贯——**3 大问题占启动问题 50%**。

## 1.2 黑/白/闪屏 3 大问题的独特性

| 独特性 | 表现 | 后果 |
|:-------|:-----|:-----|
| **用户立即感知** | 第一眼看到的就是黑/白/闪屏 | 用户立刻判定"启动失败" |
| **4 层栈穿透** | 涉及 App → WMS → SF → Display | 排查必须穿透 4 层 |
| **难复现** | 时序敏感，难复现 | 自动化测试困难 |
| **OEM 主导** | BootLogo / BootAnimation 是 OEM 决定 | 通用方案有限 |

## 1.3 行业数据

| 指标 | 数据 | 来源 |
|:-----|:-----|:-----|
| **黑屏问题占比** | 20% 启动问题 | 字节 / 阿里内部数据 |
| **白屏问题占比** | 10% 启动问题 | 字节 / 阿里内部数据 |
| **闪屏问题占比** | 5% 启动问题 | 字节 / 阿里内部数据 |
| **黑屏 P0 工单** | 30% 启动 P0 工单 | 5 大厂内部数据 |
| **冷启动 < 1s 行业基准** | < 1s | 头部 App 目标 |
| **首屏 16.67ms 阈值** | 60fps 1 帧 | Android Vitals |

> **所以呢**：黑/白/闪屏 3 大问题 = 35% 启动问题 + 30% P0 工单 = 启动期头号问题。

# 2. 边界：黑屏 vs 白屏 vs 闪屏

| 维度 | 黑屏 | 白屏 | 闪屏 |
|:-----|:-----|:-----|:-----|
| **表现** | 屏幕全黑 | 屏幕全白 | 屏幕闪烁 |
| **持续时间** | 持续 > 1s | 持续 > 500ms | 出现一次 |

| **占启动问题** | 20% | 10% | 5% |
| **解决方案** | SplashScreen | 资源预加载 | 主题统一 |

# 3. 黑屏问题：4 层栈穿透排查

## 3.1 黑屏的 5 大根因

```
   ┌────────────────────────────────────────────────────────────┐
   │  黑屏的 5 大根因（按占比排序）                              │
   └────────────────────────────────────────────────────────────┘

   1. WMS 未 ready（30%）
      └─ 表现：卡在 Boot Logo / 启动后无焦点
      └─ 原因：WindowManagerService 启动慢 / 卡死

   2. SurfaceFlinger 卡死（25%）
      └─ 表现：卡在 Boot Animation / 启动后黑屏
      └─ 原因：SF 初始化慢 / Display HAL 卡 / Buffer 队列满

   3. Launcher onCreate 卡（20%）
      └─ 表现：启动后黑屏但能输入触摸
      └─ 原因：onCreate 主线程 IO / GC 卡顿

   4. AMS 未 startHome（15%）
      └─ 表现：启动后一直黑屏，无任何窗口
      └─ 原因：AMS 启动失败 / Launcher Activity 不存在

   5. GPU 渲染卡（10%）
      └─ 表现：黑屏 + GPU 驱动报错
      └─ 原因：GPU 驱动 BUG / RenderThread 卡死
```

## 3.2 黑屏 4 步取证法

```bash
# Step 1: 看焦点窗口（WMS 状态）
adb shell dumpsys window | grep -A 2 "mCurrentFocus"
# 异常：mCurrentFocus=null → WMS 未 ready

# Step 2: 看 Window 是否有 Surface
adb shell dumpsys window windows | grep -A 5 "mHasSurface"
# 异常：mHasSurface=false → 没有显示 surface

# Step 3: 看 SurfaceFlinger layer
adb shell dumpsys SurfaceFlinger | grep -A 5 "Visible"
# 异常：layer 数 < 5 → SF 未 ready

# Step 4: 看 Display 状态
adb shell dumpsys display | head -30
# 异常：Display 状态 OFF → Display HAL 卡
```

## 3.3 黑屏实战排查剧本

### 剧本 1：WMS 未 ready

**症状**：
- 卡在 Boot Logo > 5s
- `dumpsys window` 显示 `mCurrentFocus=null`

**根因**：
- WindowManagerService 启动慢
- WMS 构造时某个依赖服务卡死
- WMS 等待 Display 设备就绪

**排查命令**：
```bash
# 1. 看 WMS 启动耗时
adb shell dumpsys activity processes | grep "WindowManager"
# 异常：WindowManagerService 未启动

# 2. 看 WMS 状态
adb shell dumpsys window | head -50
# 关键：看 WMS 启动状态

# 3. 看 SystemServer 启动状态
adb shell dumpsys bootstat | grep "boot_system_server"
# 异常：boot_system_server_time > 15s
```

**解决方案**：
- 检查 WMS 启动卡在哪一阶段
- 检查 WMS 依赖服务（InputManager / DisplayManager）状态
- 用 perfetto boot trace 定位

### 剧本 2：SurfaceFlinger 卡死

**症状**：
- 卡在 Boot Animation > 5s
- 启动后黑屏
- `dumpsys SurfaceFlinger` 异常

**根因**：
- SF 初始化慢（GPU 驱动加载慢）
- Display HAL 卡死（屏驱动问题）
- Buffer 队列满（App 提交 buffer 失败）

**排查命令**：
```bash
# 1. 看 SF 状态
adb shell dumpsys SurfaceFlinger | head -50
# 异常：layer 数 < 5 / Buffer 队列空

# 2. 看 Display HAL
adb shell logcat -d -s HwBinder:V DisplayService:V
# 关键：找 HAL error

# 3. 看 GPU 状态
adb shell logcat -d -s GPU:V SurfaceFlinger:V
# 关键：找 GPU error

# 4. 看 Buffer 队列延迟
adb shell dumpsys SurfaceFlinger --latency
# 异常：VSYNC 间隔 > 16.67ms
```

**解决方案**：
- 优化 SF 启动（关闭不必要 SF 配置）
- 修复 Display HAL（厂商驱动）
- 减少 App 提交 buffer（降低 layer 数）

### 剧本 3：Launcher onCreate 卡

**症状**：
- 启动后黑屏但能输入触摸
- `dumpsys window` 显示 Launcher 窗口但无 Surface
- traces.txt 显示主线程卡在 onCreate

**根因**：
- onCreate 主线程 IO
- onCreate GC 卡顿
- onCreate 复杂计算

**排查命令**：
```bash
# 1. 看 Launcher 进程状态
adb shell ps -A | grep launcher

# 2. 看 Launcher traces
adb shell cat /data/anr/traces.txt | grep -A 30 "launcher"
# 关键：看主线程 stack

# 3. 看 GC
adb shell logcat -d -s art:V | grep -A 2 "GC"
# 异常：启动期 GC 暂停 > 100ms

# 4. 看 Launcher 启动耗时
adb shell dumpsys activity activities | grep "launcher"
```

**解决方案**：
- onCreate 异步化（详见 B02 §6.2）
- SplashScreen 提前显示
- 减少 onCreate 资源加载

### 剧本 4：AMS 未 startHome

**症状**：
- 启动后一直黑屏，无任何窗口
- `dumpsys activity activities` 无 mResumedActivity

**根因**：
- AMS 启动失败
- Launcher Activity 不存在
- Intent 解析失败

**排查命令**：
```bash
# 1. 看 AMS 状态
adb shell dumpsys activity activities | head -50
# 异常：mResumedActivity=null / ActivityRecord 数 < 2

# 2. 看 Launcher Activity
adb shell cmd package resolve-activity --brief -c android.intent.category.HOME
# 异常：无输出 → Launcher Activity 不存在

# 3. 看 Boot 进度
adb shell dumpsys activity processes | grep "Boot"
```

**解决方案**：
- 修复 Launcher Activity 声明
- 检查 Intent 解析
- 检查 CATEGORY_HOME 注册

### 剧本 5：GPU 渲染卡

**症状**：
- 黑屏 + GPU 驱动报错
- logcat 显示 GPU error / EGL error

**根因**：
- GPU 驱动 BUG
- RenderThread 卡死
- EGL 初始化失败

**排查命令**：
```bash
# 1. 看 GPU 状态
adb shell logcat -d -s GPU:V EGL:V RenderThread:V
# 关键：找 GPU error

# 2. 看 RenderThread
adb shell dumpsys gfxinfo system_server framestats
# 异常：Janky frames > 10%

# 3. 看 GPU 渲染配置
adb shell dumpsys SurfaceFlinger | grep -A 5 "RenderEngine"
```

**解决方案**：
- 修复 GPU 驱动（厂商）
- 关闭硬件加速（fallback）
- 简化 GPU 渲染

## 3.4 黑屏根治方案：SplashScreen API

```xml
<!-- AndroidManifest.xml -->
<activity android:name=".MainActivity"
          android:theme="@style/Theme.App.Starting">
</activity>

<!-- res/values/themes.xml -->
<style name="Theme.App.Starting" parent="Theme.SplashScreen">
    <item name="windowSplashScreenBackground">@color/brand_primary</item>
    <item name="windowSplashScreenAnimatedIcon">@drawable/ic_launcher_foreground</item>
    <item name="postSplashScreenTheme">@style/Theme.App</item>
</style>
```

**原理**：
- 启动时立即显示 SplashScreen 主题
- 避免"启动后无任何画面"的黑屏
- onCreate 完成后切换到 postSplashScreenTheme

**效果**：
- 根治 90% 黑屏问题
- 用户体验提升（看到品牌图）

> **所以呢**：SplashScreen API 是 AOSP 12+ 提供的"黑屏根治方案"——必须用。

# 4. 白屏问题：资源加载缺失

## 4.1 白屏的 4 大根因

```
   ┌────────────────────────────────────────────────────────────┐
   │  白屏的 4 大根因（按占比排序）                              │
   └────────────────────────────────────────────────────────────┘

   1. 主题背景未设置（35%）
      └─ 表现：启动后白屏 > 500ms
      └─ 原因：App 主题未设置 windowBackground

   2. 资源未加载（25%）
      └─ 表现：白屏 + logcat 资源加载失败
      └─ 原因：图片 / 字符串 / 布局资源缺失

   3. View 树复杂（20%）
      └─ 表现：白屏 + onCreate 慢
      └─ 原因：measure + layout 耗时 > 500ms

   4. 启动主题错误（20%）
      └─ 表现：白屏 + 启动后切换主题
      └─ 原因：启动主题与正常主题不一致
```

## 4.2 白屏 4 步取证法

```bash
# Step 1: 看启动主题
adb shell dumpsys activity activities | grep -A 5 "theme"
# 关键：看启动 Activity 的 theme

# Step 2: 看资源加载
adb shell logcat -d -s ResourcesManager:V
# 异常：资源加载失败

# Step 3: 看 View 树
adb shell uiautomator dump /sdcard/ui.xml
adb pull /sdcard/ui.xml /tmp/
# 异常：<node ... text="" /> → View 树为空

# Step 4: 看 onCreate 耗时
adb shell logcat -d -s ActivityTaskManager:V | grep "Displayed"
# 异常：Displayed 时间 > 500ms
```

## 4.3 白屏解决方案

### 方案 1：设置 windowBackground

```xml
<!-- res/values/themes.xml -->
<style name="Theme.App" parent="Theme.Material3.DayNight">
    <!-- 🟢 关键：设置窗口背景 -->
    <item name="android:windowBackground">@color/brand_primary</item>
    <item name="android:colorBackground">@color/brand_primary</item>
</style>
```

### 方案 2：使用 SplashScreen API

```xml
<!-- 详见 §3.4 -->
<style name="Theme.App.Starting" parent="Theme.SplashScreen">
    <item name="windowSplashScreenBackground">@color/brand_primary</item>
    <item name="windowSplashScreenAnimatedIcon">@drawable/ic_launcher_foreground</item>
</style>
```

### 方案 3：减少 onCreate 耗时

```java
// 优化前：onCreate 加载大量资源
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    loadBigImage();      // 500ms
    loadBigLayout();     // 200ms
    initDatabase();      // 300ms
}

// 优化后：onCreate 异步化
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    
    // 🟢 异步加载
    CompletableFuture.runAsync(() -> {
        loadBigImage();
        loadBigLayout();
        initDatabase();
    });
}
```

> **所以呢**：白屏 90% 是 windowBackground 未设置——**一行 XML 解决**。

# 5. 闪屏问题：主题切换不连贯

## 5.1 闪屏的 3 大根因

```
   ┌────────────────────────────────────────────────────────────┐
   │  闪屏的 3 大根因                                            │
   └────────────────────────────────────────────────────────────┘

   1. 启动主题与正常主题不一致（40%）
      └─ 表现：启动主题 → 正常主题 切换瞬间闪烁
      └─ 原因：postSplashScreenTheme 设置错误

   2. windowBackground 资源加载（30%）
      └─ 表现：背景色或图片闪烁
      └─ 原因：资源未预加载 / 主题切换资源不一致

   3. BootAnimation 与 Launcher 切换（30%）
      └─ 表现：BootAnimation 结束后 → Launcher 黑屏瞬间
      └─ 原因：BootAnimation 与 Launcher 主题不一致
```

## 5.2 闪屏 4 步取证法

```bash
# Step 1: 看启动主题
adb shell dumpsys activity activities | grep -A 5 "theme"

# Step 2: 看 windowBackground 切换日志
adb shell logcat -d -s WindowManager:V | grep "Background"

# Step 3: 看 BootAnimation 结束时间
adb shell logcat -d -s bootanim:V

# Step 4: 看 Launcher 第一帧时间
adb shell logcat -d -s ActivityTaskManager:V | grep "Displayed"
# 异常：Displayed 时间 > BootAnimation 结束时间 + 100ms → 闪屏
```

## 5.3 闪屏解决方案

### 方案 1：统一启动主题与正常主题

```xml
<!-- res/values/themes.xml -->
<style name="Theme.App" parent="Theme.Material3.DayNight">
    <!-- 统一背景色 -->
    <item name="android:windowBackground">@color/brand_primary</item>
</style>

<style name="Theme.App.Starting" parent="Theme.SplashScreen">
    <!-- 与 Theme.App 保持一致 -->
    <item name="windowSplashScreenBackground">@color/brand_primary</item>
    <item name="windowSplashScreenAnimatedIcon">@drawable/ic_launcher_foreground</item>
    <item name="postSplashScreenTheme">@style/Theme.App</item>
</style>
```

### 方案 2：使用 SplashScreen 退出动画

```java
// MainActivity.java
public class MainActivity extends Activity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        SplashScreen splashScreen = getSplashScreen();
        splashScreen.setOnExitAnimationListener(splashView -> {
            // 启动完成后的动画
            ObjectAnimator animator = ObjectAnimator.ofPropertyValuesHolder(
                splashView,
                PropertyValuesHolder.ofFloat(View.ALPHA, 0f)
            );
            animator.addListener(new AnimatorListenerAdapter() {
                @Override
                public void onAnimationEnd(Animator animation) {
                    splashView.remove();
                }
            });
            animator.start();
        });
        
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
    }
}
```

### 方案 3：BootAnimation 与 Launcher 主题一致

```bash
# 1. 看 BootAnimation 配置
adb shell cat /system/media/bootanimation/desc.txt
# 关键：看 BootAnimation 资源

# 2. 看 BootAnimation 退出时间
adb shell logcat -d -s bootanim:V

# 3. 看 Launcher 第一帧时间
adb shell logcat -d -s ActivityTaskManager:V | grep "Displayed"
# 关键：BootAnimation 结束时间 < Launcher 第一帧时间 → 无缝切换
```

# 6. 风险地图（与 Stability S05 联动 · 强制）

> **本节是 v4 强制要求**——黑/白/闪屏是稳定性 S05 启动期核心场景。

## 6.1 黑屏 5 大风险（与 S05 联动）

| 风险 | 触发条件 | Watchdog 兜底 | 怎么查 |
|:-----|:---------|:-------------|:------|
| **WMS 未 ready** | WMS 启动 > 5s | ✅ 30s 杀 | `dumpsys window` |
| **SF 卡死** | SF 启动 > 5s | ✅ 30s 杀 | `dumpsys SurfaceFlinger` |
| **Launcher 卡** | onCreate > 5s | ✅ 30s 杀 | traces.txt |
| **AMS 未 startHome** | AMS 启动 > 5s | ✅ 30s 杀 | `dumpsys activity` |
| **GPU 卡** | GPU 驱动 BUG | ❌ | logcat GPU |

## 6.2 白屏 4 大风险

| 风险 | 触发条件 | 怎么查 | 解决方案 |
|:-----|:---------|:------|:---------|
| **未设置 windowBackground** | 主题未配置 | 启动主题 | 设置 windowBackground |
| **资源未加载** | 图片/字符串缺失 | logcat ResourcesManager | 资源预加载 |
| **View 树复杂** | onCreate 慢 | traces.txt | onCreate 异步化 |
| **启动主题错误** | postSplashScreenTheme 错 | dumpsys activity | 主题统一 |

## 6.3 闪屏 3 大风险

| 风险 | 触发条件 | 怎么查 | 解决方案 |
|:-----|:---------|:------|:---------|
| **主题不一致** | 启动主题 ≠ 正常主题 | dumpsys activity | 主题统一 |
| **windowBackground 切换** | 主题切换资源不一致 | logcat WindowManager | 资源预加载 |
| **BootAnimation 切换** | BootAnimation 退出 ≠ Launcher 进入 | logcat bootanim | 时序对齐 |

## 6.4 启动期视觉问题严重程度

| 等级 | 现象 | 用户感知 | 工单优先级 |
|:-----|:-----|:---------|:----------|
| **P0** | 启动后一直黑屏 | 🔴 立刻感知 | 🔴 立即响应 |
| **P0** | 启动后白屏 > 1s | 🔴 立刻感知 | 🔴 立即响应 |
| **P1** | 闪屏 > 200ms | 🟡 偶尔感知 | 🟡 当天响应 |
| **P2** | 闪屏 < 200ms | 🟢 几乎不可感知 | 🟢 1 周内响应 |

# 7. 5 实战案例

## 7.1 案例 1：某手机启动后黑屏 3s

**症状**：
- 启动后黑屏 3s，然后显示 Launcher
- `dumpsys window` 显示 Launcher Activity 已 ready

**根因**：
- onCreate 主线程 SharedPreferences IO 1.5s
- onCreate 主线程 数据库查询 1.5s

**排查过程**：
```bash
# 1. 看 traces.txt
adb shell cat /data/anr/traces.txt | grep -A 30 "launcher"
# 关键：看主线程 stack

# 2. 看 onCreate 耗时
adb shell logcat -d -s ActivityTaskManager:V | grep "Displayed"
# 异常：Displayed 时间 > 2s

# 3. 看 GC
adb shell logcat -d -s art:V | grep "GC"
# 异常：启动期 GC 暂停 > 100ms
```

**解决方案**：
- onCreate 异步化（SharedPreferences + 数据库查询放后台）
- SplashScreen 提前显示
- 优化后：黑屏 0s（SplashScreen 立即显示）

**收益**：黑屏 3s → 0s（100% 解决）

## 7.2 案例 2：某 App 启动后白屏 500ms

**症状**：
- 启动后白屏 500ms，然后显示 App 内容
- `dumpsys activity` 显示 App Activity 已 ready

**根因**：
- App 主题未设置 `windowBackground`
- 启动后默认背景为白色

**排查过程**：
```bash
# 1. 看启动主题
adb shell dumpsys activity activities | grep -A 5 "theme"
# 异常：theme 不含 windowBackground

# 2. 看资源加载
adb shell logcat -d -s ResourcesManager:V
# 关键：找资源加载失败
```

**解决方案**：
```xml
<style name="Theme.App" parent="Theme.Material3.DayNight">
    <item name="android:windowBackground">@color/brand_primary</item>
</style>
```

**收益**：白屏 500ms → 0s（100% 解决）

## 7.3 案例 3：某设备启动闪屏 200ms

**症状**：
- 启动后闪屏 200ms（颜色切换）
- `dumpsys activity` 显示主题切换

**根因**：
- 启动主题为 `Theme.SplashScreen`（背景白）
- postSplashScreenTheme 为 `Theme.App`（背景蓝）
- 切换时颜色不一致

**排查过程**：
```bash
# 1. 看启动主题
adb shell dumpsys activity activities | grep -A 5 "theme"
# 异常：theme 切换

# 2. 看 BootAnimation 结束时间
adb shell logcat -d -s bootanim:V

# 3. 看 Launcher 第一帧时间
adb shell logcat -d -s ActivityTaskManager:V | grep "Displayed"
```

**解决方案**：
```xml
<style name="Theme.App.Starting" parent="Theme.SplashScreen">
    <item name="windowSplashScreenBackground">@color/brand_primary</item>
    <item name="postSplashScreenTheme">@style/Theme.App</item>
</style>
```

**收益**：闪屏 200ms → 0s（100% 解决）

## 7.4 案例 4：某电视启动后黑屏 10s

**症状**：
- 启动后黑屏 10s
- `dumpsys SurfaceFlinger` 显示 layer 数 < 5

**根因**：
- Display HAL 启动慢（OEM 定制）
- SF 等待 Display HAL 就绪

**排查过程**：
```bash
# 1. 看 SF 状态
adb shell dumpsys SurfaceFlinger | head -50
# 异常：layer 数 < 5

# 2. 看 Display HAL
adb shell logcat -d -s HwBinder:V DisplayService:V
# 关键：找 HAL error

# 3. 看 BootAnimation
adb shell logcat -d -s bootanim:V
# 异常：bootanim 启动慢
```

**解决方案**：
- 优化 Display HAL 启动（OEM 主导）
- 优化 SF 启动（关闭不必要 SF 配置）
- 优化后：黑屏 10s → 1s

**收益**：黑屏 10s → 1s（90% 解决）

## 7.5 案例 5：某应用启动卡在桌面 5s

**症状**：
- 启动后显示桌面 5s
- `dumpsys window` 显示 Launcher 已 ready

**根因**：
- Launcher 加载大量 widget
- 启动后 widget 渲染卡 5s

**排查过程**：
```bash
# 1. 看 widget 数量
adb shell dumpsys appwidget | head -30

# 2. 看 widget 渲染
adb shell logcat -d -s AppWidget:V

# 3. 看 GPU 渲染
adb shell dumpsys gfxinfo com.android.launcher3 framestats
# 异常：Janky frames > 20%
```

**解决方案**：
- 减少 widget 数量
- 异步加载 widget
- 优化 widget 渲染

**收益**：卡顿 5s → 1s（80% 解决）

# 8. dumpsys 怎么取证（与 Dumpsys D03/D05 联动 · 强制）

## 8.1 黑/白/闪屏 4 步取证法

| Step | 命令 | 目的 |
|:-----|:-----|:-----|
| 1 | `adb shell dumpsys window \| grep mCurrentFocus` | 看 WMS 状态 |
| 2 | `adb shell dumpsys SurfaceFlinger` | 看 SF 状态 |
| 3 | `adb shell dumpsys activity activities` | 看 Activity 状态 |
| 4 | `adb shell logcat -d -s ActivityTaskManager:V \| grep Displayed` | 看第一帧时间 |

## 8.2 黑屏取证脚本

```bash
# 场景：启动后黑屏
# 步骤 1: 看焦点窗口
adb shell dumpsys window | grep "mCurrentFocus"
# 异常：mCurrentFocus=null → WMS 未 ready

# 步骤 2: 看 Window 是否有 Surface
adb shell dumpsys window windows | grep -A 5 "mHasSurface"
# 异常：mHasSurface=false → 没显示

# 步骤 3: 看 SurfaceFlinger layer
adb shell dumpsys SurfaceFlinger | grep -A 5 "Visible"
# 异常：layer 数 < 5 → SF 未 ready

# 步骤 4: 看 Display 状态
adb shell dumpsys display | head -30
# 异常：Display 状态 OFF → Display HAL 卡
```

## 8.3 白屏取证脚本

```bash
# 场景：启动后白屏
# 步骤 1: 看启动主题
adb shell dumpsys activity activities | grep -A 5 "theme"
# 异常：theme 不含 windowBackground

# 步骤 2: 看资源加载
adb shell logcat -d -s ResourcesManager:V
# 异常：资源加载失败

# 步骤 3: 看 View 树
adb shell uiautomator dump /sdcard/ui.xml
# 异常：<node ... text="" /> → View 树为空

# 步骤 4: 看 onCreate 耗时
adb shell logcat -d -s ActivityTaskManager:V | grep "Displayed"
# 异常：Displayed 时间 > 500ms
```

## 8.4 闪屏取证脚本

```bash
# 场景：启动闪屏
# 步骤 1: 看启动主题
adb shell dumpsys activity activities | grep -A 5 "theme"

# 步骤 2: 看 windowBackground 切换日志
adb shell logcat -d -s WindowManager:V | grep "Background"

# 步骤 3: 看 BootAnimation 结束时间
adb shell logcat -d -s bootanim:V

# 步骤 4: 看 Launcher 第一帧时间
adb shell logcat -d -s ActivityTaskManager:V | grep "Displayed"
```
