## 2.2 关键问题

| 问题 | 现状 | 影响 |
|:-----|:-----|:-----|
| 崩溃率 0.15% | 行业基线 0.05% | **3 倍恶化** |

## 2.3 根因分析

- **崩溃**：Top 10 崩溃堆栈占 60% 崩溃量
- **ANR**：主线程同步 IO 占 70%
- **启动慢**：Application.onCreate 同步 5s+

# 5. 治理阶段 3（M4：1 个月，ANR 治理）

## 5.1 动作

| 动作 | 完成时间 | owner |
|:-----|:---------|:------|
| 主线程同步 IO 异步化 | 2 周 | 业务团队 |
| Input ANR 专项（占 60%）| 1 周 | Framework |
| Service / Broadcast ANR 专项 | 1 周 | Framework |

## 5.2 关键代码改动

```java
// 改前：主线程读文件
public void onCreate() {
    super.onCreate();
    FileInputStream fis = new FileInputStream(getFilesDir() + "/config.json");  // ❌ 200ms
}

// 改后：异步 + 缓存
public void onCreate() {
    super.onCreate();
    new Thread(() -> {
        Config config = loadConfig();
        runOnUiThread(() -> applyConfig(config));
    }).start();
}
```

## 5.3 关键数字

- ANR 率：0.18% → 0.08%
- Stability Score（ANR 分）：64 → 84
- 整体 Score：80 → 85（+5 分）

# 6. 治理阶段 4（M5：1 个月，性能治理）

## 6.1 动作

| 动作 | 完成时间 | owner |
|:-----|:---------|:------|
| Application.onCreate 异步化 | 1 周 | 启动团队 |
| 启动期懒加载 | 2 周 | 业务团队 |
| 首页渲染优化 | 1 周 | UI 团队 |

## 6.2 启动期优化

```
改前（启动链）：
Process.start → Application.onCreate (5s 同步) → MainActivity.onCreate → 第一帧
                                                            ↑
                                                      总耗时 5.5s

改后（启动链）：
Process.start → Application.onCreate (50ms 异步) → MainActivity.onCreate (200ms 懒加载) → 第一帧
                                                                  ↑
                                                            总耗时 1.8s
```

## 6.3 关键数字

- 启动 P95：3.5s → 1.8s
- Stability Score（性能分）：60 → 88
- 整体 Score：85 → 90（+5 分）
