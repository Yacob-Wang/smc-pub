# 3. Layer 2：ClickHouse 存储层

## 3.1 Schema 设计

```sql
CREATE TABLE events_raw (
    event_id String,
    event_type LowCardinality(String),  -- 'crash'/'anr'/'startup'/'frame'
    session_id String,
    user_id String,
    device_id String,
    app_version String,
    os_version String,
    device_model String,
    timestamp DateTime64,
    payload String
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(timestamp)
ORDER BY (event_type, app_version, timestamp)
TTL timestamp + INTERVAL 365 DAY;
```

## 3.2 物化视图（5 大度量）

```sql
-- 崩溃率物化视图
CREATE MATERIALIZED VIEW crash_rate_mv
ENGINE = SummingMergeTree()
ORDER BY (app_version, hour)
AS SELECT
    app_version,
    toStartOfHour(timestamp) AS hour,
    countIf(event_type = 'je_exception' AND session_id != '') AS je_sessions,
    countIf(event_type = 'session_start') AS total_sessions
FROM events_raw
WHERE event_type IN ('je_exception', 'session_start')
GROUP BY app_version, hour;
```

## 3.3 查询性能

| 查询 | 响应时间 |
|:-----|:---------|
| 单维度 P99 崩溃率（30 天）| < 1s |
| 多维度聚合（5 维 × 30 天）| < 3s |
| Top 10 崩溃堆栈（30 天）| < 5s |
| 用户分群查询 | < 10s |

# 4. Layer 3：Flink 计算层

## 4.1 4 个核心 Job

```java
// Job 1: 5 大度量聚合
DataStream<RawEvent> events = ...;
events
    .keyBy(e -> e.appVersion)
    .window(TumblingEventTimeWindows.of(Time.minutes(1)))
    .aggregate(new CrashRateAggregator())
    .sink(new ClickHouseSink<>("crash_rate_metrics"));

// Job 2: Stability Score 计算
metricsStream
    .keyBy(m -> m.appVersion)
    .window(TumblingEventTimeWindows.of(Time.hours(1)))
    .process(new StabilityScoreCalculator())
    .sink(new ClickHouseSink<>("stability_score"));

// Job 3: 异常检测
metricsStream
    .keyBy(m -> m.appVersion)
    .process(new AnomalyDetector())
    .filter(e -> e.isAnomaly)
    .sink(new KafkaSink<>("apm_alerts"));

// Job 4: 告警路由
alertStream
    .keyBy(a -> a.severity)
    .process(new AlertRouter())
    .sink(new WebhookSink<>("alert_dispatcher"));
```

## 4.2 异常检测算法

```python
def detect_anomaly(current, baseline, threshold=3):
    """时序异常检测：3-sigma"""
    if abs(current - baseline.mean) > threshold * baseline.std:
        return True
    return False
```

详见 [05-Governance/AI-Native/03_AI_for_Stability/F02-时序异常检测.md](../AI-Native/03_AI_for_Stability/F02-时序异常检测.md)。

# 6. 数据生命周期

## 6.1 三级存储

| 级别 | 存储 | 保留期 | 成本 |
|:-----|:-----|:-------|:-----|
| **热**（最近 7 天）| ClickHouse SSD | 7 天 | 高 |
| **温**（8-90 天）| ClickHouse HDD | 90 天 | 中 |
| **冷**（91-365 天）| S3 / OSS | 365 天 | 低 |

## 6.2 自动迁移

```sql
-- ClickHouse TTL 自动迁移
ALTER TABLE events_raw
    MODIFY TTL
        timestamp + INTERVAL 7 DAY TO VOLUME 'cold1',
        timestamp + INTERVAL 90 DAY TO VOLUME 'cold2',
        timestamp + INTERVAL 365 DAY DELETE;
```

# 8. 8 反例

| # | 反例 | 错误做法 | 正确做法 |
|:-:|:-----|:---------|:---------|
| 1 | **不用 Kafka** | 直接写 DB | **必异步解耦** |
| 2 | **MySQL 存度量** | OLTP 存 OLAP | **ClickHouse 列存** |
| 3 | **不分区** | 单分区表 | **必按时间分区** |
| 4 | **不 TTL** | 永远不删 | **必设 TTL** |
| 5 | **同步链路** | Kafka → DB 同步 | **全异步** |
| 6 | **无冷热分层** | 全部 SSD | **热 SSD + 温 HDD** |
| 7 | **不监控后端** | 后端挂了不知道 | **加后端 APM** |
| 8 | **过度采集** | 全字段 | **按需采集** |

# 9. 5 条 Takeaway

1. **Kafka + Flink + ClickHouse 三件套** —— 业界标准
2. **Kafka 削峰填谷** —— 100MB/s 吞吐，3 节点起步
3. **ClickHouse 列存 + TTL** —— 30 天 P99 < 3s
4. **Flink 4 Job** —— 5 度量 + 综合分 + 异常 + 告警
5. **三级存储分层** —— 热 SSD + 温 HDD + 冷 S3
