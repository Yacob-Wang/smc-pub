## 三、Activity 系列大纲（9 篇）

### 系列角色
**应用层四大组件的"前台门面"**。理解 Activity 是理解 AMS 调度、WMS 焦点、Input 分发、PMS 解析的入口；启动 Activity 是 Service / Broadcast / Provider 启动的常见父调用。

   │
   ▼

   │
   ▼
A04 启动模式    ← WMS 系列(Task管理)
   │
   ▼
A05 Intent匹配  ← PMS 系列(组件解析)
   │
   ▼
A06 ConfigChange ← 横切专题（独立写）
   │

   │
   ▼
A09 内存治理    ← MM_v2 系列(Activity泄漏)

## 四、Service 系列大纲（9 篇）

### 系列角色
**后台执行单元 + 跨进程通信桥梁**。Service 是 ANR 的高发区（独立 Service ANR 阈值 20s 是 Android 7 引入的），也是 Android 12+ 后台限制（ForegroundService 强制类型化）的重灾区。

### 依赖关系

```
S01 全景        (本系列根)
   │
   ▼
S02 startService路径 ← Activity 系列(A02 启动流程)
   │
   ▼
S03 bindService路径  ← Binder 系列(跨进程)
   │
   ▼
S04 前台服务FGS ← Android 14+ 后台启动限制
   │
   ▼
S05 WorkManager演进 ← 后台任务最佳实践
   │
   ▼
S06 多客户端与死亡链路 ← Binder 系列(死亡通知)
   │
   ▼
S07 Service ANR全景 ← 风险地图
   │
   ▼
S08 进程保活与onTrimMemory ← 横切专题
   │
   ▼
S09 跨进程Binder限制与Service上限 ← 诊断治理
```

## 五、Broadcast 系列大纲（9 篇）

### 系列角色
**跨进程事件分发机制**。Broadcast 是 ANR 高发区（前台 10s / 后台 60s 阈值不同），Android 14+ 引入了 `RECEIVER_NOT_EXPORTED` / `RECEIVER_EXPORTED` 强制声明，Android 17 进一步收紧；静态注册 BOOT_COMPLETED 涉及 PMS 解析。

### 依赖关系

```
B01 全景        (本系列根)
   │
   ▼
B02 注册机制    ← PMS 系列(manifest解析)
   │
   ▼
B03 发送流程    ← Activity 系列(A02)、Service 系列(S02 父调用)
   │
   ▼
B04 有序广播    ← BroadcastQueue 串行调度
   │
   ▼
B05 粘性广播与Android 17演进 ← 历史演进(破例：演进型)
   │
   ▼
B06 LocalBroadcast/进程内替代 ← 横切专题
   │
   ▼
B07 Android 14+ 后台广播限制 ← 风险地图
   │
   ▼
B08 Broadcast ANR全景 ← 风险地图
   │
   ▼
B09 系统广播与开机广播 ← 诊断治理
```

## 九、执行顺序与里程碑

### 9.1 建议执行顺序

**第 1 批：Activity 系列（9 篇）**——因为 Service/Broadcast 都会引用 Activity（A02 启动流程是父调用）。
**第 2 批：Service 系列（9 篇）**——依赖 Activity 完成。
**第 3 批：Broadcast 系列（9 篇）**——三系列中最后。

### 9.2 里程碑

| 里程碑 | 内容 | 交付物 |
|--------|------|--------|
| M0 | **当前**：3 系列总大纲（本文件） | `三系列重写规划-2026-07-18.md` |
| M1 | Activity 系列 README + 9 篇全文 | `Android_Framework/Activity/README.md` + 9 篇 |
| M2 | Service 系列 README + 9 篇全文 | `Android_Framework/Service/README.md` + 9 篇 |

| M4 | 跨系列一致性回归 + 案例索引 + 引用矩阵 | `Reference/案例索引.md` 更新 |

### 9.3 时间预估

按 v4 节奏，每篇平均 2 轮校准，每轮 30-60 分钟：
- 9 篇 × (1 写 + 2 校准) × 45 分钟 ≈ **20 小时/系列**
- 3 系列 = **60 小时**

**建议拆分**：每轮 1-2 篇，避免一次性 27 篇导致记忆断片。
