# 版本基线（Reference · 版本基线）

> **基线**：AOSP `android-17.0.0_r1`（API 37） + Linux `android17-6.18` LTS
> **生效日期**：2026-07-18
> **覆盖系列**：Activity / Service / Broadcast / ContentProvider 四大组件

## 三、AOSP 17 关键硬变化（写文章时必须覆盖）

### 3.1 ART 17 硬变化

| 变化 | 影响范围 | 必带系列 |
|------|---------|---------|
| **分代 GC 强化** | ART 17 GC 算法 | Activity / Service / Broadcast / ContentProvider |
| **无锁 MessageQueue** | API 37+ 主线程消息 | Activity / Service / Broadcast |
| **static final 不可变** | 编译期优化 | Activity / Service / Broadcast |
| **AppFunctions 集成** | AI Agent OS | Activity / Service |
| **类去重** | 多 dex 优化 | Activity / Service |
| **PAC/BTI 集成** | 安全性 | ContentProvider |
| **Quickened Bytecode** | ART 17 优化 | Activity |
| **Crash 快速路径** | ART 17 优化 | Activity / Service / Broadcast |
| **持续性能监控 APEX** | `com.android.profiling` | Activity |
| **FastNative 强化** | JNI 优化 | Service / ContentProvider |
| **Slot Pool** | JNI 优化 | Service / ContentProvider |
| **ExceptionClear 自动检测** | JNI 优化 | Service / ContentProvider |
| **Async-Signal-Safety 强化** | SignalCatcher | Service / Broadcast |
| **pidfds 扩展** | Kernel 集成 | Service / Broadcast |
| **MAX_QUERY_RESULTS = 1000** | ContentResolver 限制 | ContentProvider |

### 3.2 Linux 6.18 LTS 硬变化

| 变化 | 影响范围 | 必带系列 |
|------|---------|---------|
| **Rust 版 Binder** | binder_alloc_rust.rs | Service / ContentProvider |
| **dm-pcache 持久内存缓存** | 内存 | n/a（本系列不涉及） |
| **sheaves 内存分配** | 内存 | n/a |
| **pidfds 扩展支持内核命名空间** | 进程 fd | Service / Broadcast |
| **eBPF 加密签名** | eBPF 验证 | n/a |
| **bcachefs 移除** | 文件系统 | n/a |
| **XFS 在线 check/repair** | 文件系统 | n/a |
| **exFAT 16x 加速** | 文件系统 | Activity（图片加载） |
| **USAP 预热池** | Zygote | Activity / ContentProvider |

### 3.3 其他硬变化

| 变化 | 引入版本 | 影响 |
|------|---------|------|
| **RECEIVER_EXPORTED 强制** | AOSP 14 (API 34) | Broadcast 必须声明 |
| **FGS 类型化** | AOSP 29 引入 / 34 强制 | Service 必须声明类型 |
| **后台启动 FGS 限制** | AOSP 26 / 14 收紧 | FGS 启动需要前台 |
| **MAX_QUERY_RESULTS** | AOSP 17 | ContentProvider query 上限 1000 |
| **BROADCAST_FG_LONG_TIMEOUT** | AOSP 17 | 长广播时限 60s |
| **BROADCAST_BG_LONG_TIMEOUT** | AOSP 17 | 长广播时限 120s |
| **SHORT_SERVICE** | AOSP 17 | 短期服务 ≤ 3 分钟 |
| **`<queries>` 强制** | AOSP 11+ | 包可见性 |

### 5.2 校对状态

每篇附录 B 必须标注每个路径的校对状态：

| 状态 | 含义 |
|------|------|
| **已校对** | 在 AOSP 17 上确认存在 |

## 七、版本升级流程

### 7.1 升级前评估

```
步骤 1：基线评估
  - 新版本相对旧版本的关键 API 变化（cs.android.com diff）
  - 哪些系列受影响（按引用矩阵过滤）
  - 评估工作量（篇数 × 平均改 30% 章节）

步骤 2：批次升级
  - 优先升级引用次数最多的文章（影响面最大）
  - 不在主线的系列可以延后
```

### 7.2 升级执行

```
步骤 3：跨系列一致性回归
  - 升级后跑一遍 v4 §4 质量清单（26 项）
  - 重点检查：版本基线声明、跨篇引用是否还指向存在的文章

步骤 4：基线切换公告
  - 在 README 顶部加 "本系列基线：android-X（自 YYYY-MM-DD 起）"
  - 在术语表/版本基线登记表登记
```

### 7.3 反模式

- ❌ 一次性把所有系列基线全切
- ❌ 50% 文章是 A14，50% 是 A17，读者困惑
- ❌ 不更新引用矩阵就升级
