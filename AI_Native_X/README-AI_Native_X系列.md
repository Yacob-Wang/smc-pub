# AI Native X 系列（v3 新增 · P0）

> **本系列是 v3 路线图（JD 锚定版）的核心新增**。
>
> 锚定 JD：「软件系统设计架构师 · 65-80K · 16 薪 · 5-10 年」
> 触发需求：JD 职责 3「探索和推进端侧 AI、大模型等前沿智能技术与 Android/OS 底层框架的系统级融合，设计并构建下一代'AI OS'智能化系统架构」+ 职位要求 3「AI/ML 理论基础 + 主流框架 + **端侧推理引擎**」+ 加分项 3「**AI 加速器（NPU/GPU/DSP）** 驱动开发或优化经验者优先」

---

## 为什么 AI 必须独立成 3 个子系列

**单一系列承载不了"AI Native OS"这个目标**。JD 第 3 条要求的不是"懂 AI"，是"**用 AI 重新设计 Android 底层架构**"——这至少包含 3 个独立的技术栈：

```
                     ┌─────────────────────────────────────────────┐
                     │  02 AI_Native_OS  （操作系统级 AI 架构）       │
                     │  AI Agent OS / 系统级 AI 服务 / 智能化框架    │
                     │  ★ 简历上对应"主导下一代 AI OS 架构设计"        │
                     └──────────────────┬──────────────────────────┘
                                        │  调用 / 编排
                                        ▼
                     ┌─────────────────────────────────────────────┐
                     │  01 AI_Native_Runtime  （端侧 AI 基础设施）   │
                     │  HAL AI / NNAPI / TFLite / 加速器 / 端侧 LLM │
                     │  ★ 简历上对应"主导端侧推理引擎落地"            │
                     └──────────────────┬──────────────────────────┘
                                        │  治理 / 优化
                                        ▼
                     ┌─────────────────────────────────────────────┐
                     │  03 AI_for_Stability  （AI 治理稳定性）       │
                     │  AI 预测 ANR / 智能归因 / 异常检测             │
                     │  ★ 简历上对应"主导 AI 驱动的稳定性治理"        │
                     └─────────────────────────────────────────────┘
```

| 子系列 | 定位 | 核心技术栈 | 篇数 | 攻破时段 |
|---|---|---|---|---|
| **01 AI_Native_Runtime** | 机制层：AI 怎么在端侧跑起来 | HAL AI / NNAPI / TFLite / ONNX / GPU Delegate / NPU / 端侧 LLM | 8 篇 | 2026 H2 |
| **02 AI_Native_OS** | 架构层：AI 怎么重塑操作系统 | Android System Intelligence / AICore / AI Agent OS / 端侧 LLM 系统集成 | 6 篇 | 2027 H1 |
| **03 AI_for_Stability** | 应用层：AI 怎么赋能稳定性 | 时序异常检测 / 智能归因 / AI 预测 ANR / LLM 解读 logs | 6 篇 | 2027 H2 |

**总计 20 篇**，3 年内完成。
