# GC 系统（ART GC 9 篇系列）

> **本目录是 ART 大模块下，GC 系统的完整专题**——从理论到实战，9 篇独立成体系。
>
> **完全取代** `_archive/v1_04-内存与GC` 的旧内容（保留作为参考）。

---

## 一句话定位

**ART 的 GC 系统 = 负责 Java/Kotlin 对象分配 + 回收的完整子系统**。
所有 OOM / 卡顿 / 内存泄漏 / Finalizer 超时 / Hook 框架兼容性 / Native 内存反哺……只要涉及 Java 堆的，最终都会回到 GC 系统的某个机制。

## 工程默认值基线（关键可调参数一览）

| 参数 | 典型默认 | 选用准则 | 踩坑提醒 |
|------|---------|---------|---------|
| `dalvik.vm.heapgrowthlimit` | 256MB | 默认即可 | 误用 `largeHeap` 被 LMK 杀得更快 |
| `dalvik.vm.heapsize` | 512MB | 仅 `largeHeap=true` 生效 | 误用会让 GC 扫描更慢 |
| `dalvik.vm.softrefthreshold` | 4 (1/4) | 调小 → SoftRef 保留更少 | 影响 Glide 缓存命中率 |
| `dalvik.vm.heaptargetutilization` | 0.75 | 调小 → 堆更早收缩 | 太低会触发频繁 Trim |
| `dalvik.vm.gc.max-relative-concurrent-start-threshold` | 0.05 | 调整 CC GC 启动时机 | 影响后台 GC 频率 |
| CC GC 读屏障模式 | 全部开启 | 高版本默认 | 关闭会引发内存错误 |

## 写作约定

每篇遵循统一的目录结构（本指南）：

```
[篇号]-[专题]/
├── README.md                   # 本篇目录索引（章节速览 + 跨篇引用）
├── [篇号]-01-[章节标题].md     # 各章节文章
├── [篇号]-02-[章节标题].md
├── ...
├── [篇号]-案例-[N]-[标题].md    # 实战案例
└── appendix/                    # 4 个附录
    ├── A-源码索引.md
    ├── B-路径对账.md
    └── D-工程基线.md
```

每篇最后一节强制包含：
1. **跨篇引用**——本篇被谁引用 / 引用了谁
2. **下篇预告**——下一篇的关键看点
