## 一、APM 概述

### 9.10.1 APM 是什么

```
APM（Application Performance Monitoring）：

- 应用性能监控
- 监控 App 的运行状态
- 包括：性能 / 错误 / 网络 / 用户行为等
- GC 监控是 APM 的重要模块
-【AOSP 17 增强】GC 扩展元数据集成
-【AOSP 17 增强】ObjectFree 事件集成
-【AOSP 17 增强】sheaves slab Native 堆分类
```

### 9.10.2 自建 APM vs SaaS APM（AOSP 17 增强）

| 维度 | 自建 APM | SaaS APM |
|:---|:---|:---|
| **数据安全** | 数据在自己的服务器 | 数据在第三方 |
| **定制能力** | 强（含 AOSP 17 增强） | 弱（受限于 SDK 能力） |
| **AOSP 17 适配** | 完全自主适配 | 部分 SaaS 还未适配 |
| **成本** | 长期低（前期高） | 持续付费 |
| **维护** | 需要团队维护 | 由第三方维护 |
| **集成深度** | 可深度集成（含 JVMTI 原生库） | 受限于 SDK 能力 |

```java
    private ArtInternalStateCollector artInternalStateCollector;  //【AOSP 17 新增】

    private AlertManager alertManager;

        return INSTANCE;
    }
    
    public void init(Context context, Config config) {
        // 1. 初始化上报器
        reporter = new Reporter(config.getServerUrl(), config.getAppId());
        
        // 2. 初始化告警
        alertManager = new AlertManager(reporter, config.getAlertRules());
        
        // 3. 初始化采集器
        gcCollector = new GcCollector(alertManager);
        memoryCollector = new MemoryCollector(reporter);
        //【AOSP 17 新增】
        artInternalStateCollector = new ArtInternalStateCollector(reporter);
        sheavesCollector = new SheavesCollector(reporter);
        
        // 4. 启动 JVMTI 监听（Android 8+）
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            JvmtiGcMonitor.enableGcEvents(gcCollector);
        }
        
        //【AOSP 17 新增】启动 ObjectFree 监听（Android 11+）
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
            JvmtiGcMonitor.enableObjectFreeEvents(gcCollector);
        }
        
        // 5. 启动内存采集
        memoryCollector.start();
        //【AOSP 17 新增】
        artInternalStateCollector.start();
        sheavesCollector.start();
    }
}
```

## 四、GC 数据采集（AOSP 17 增强）

### 9.10.8 GcCollector 实现（AOSP 17 增强）

```java
public class GcCollector {
    private final AlertManager alertManager;
    
    // 累计指标
    private long gcCount = 0;
    private long totalPauseTime = 0;
    private long maxPauseTime = 0;
    private final List<Long> pauseTimeList = new ArrayList<>();
    
    //【AOSP 17 新增】分代 GC 统计
    private long youngGcCount = 0;
    private long oldGcCount = 0;
    private long stickyGcCount = 0;
    private long totalPromotedObjects = 0;
    
    //【AOSP 17 新增】软阈值统计
    private long softThresholdReached = 0;
    private long hardThresholdReached = 0;
    
    //【AOSP 17 新增】ObjectFree 统计
    private long objectFreeCount = 0;
    private final Set<String> freedClassNames = new HashSet<>();
    
    public GcCollector(AlertManager alertManager) {
        this.alertManager = alertManager;
    }
    
    // JVMTI 回调
    public void onGcStart() {
        gcStartTime = System.currentTimeMillis();
    }
    
    //【AOSP 17 增强】GC 结束回调（含扩展元数据）
    public void onGcFinish(jvmtiGarbageCollectionInfo gcInfo) {
        long pauseTime = System.currentTimeMillis() - gcStartTime;
        
        // 更新累计指标
        gcCount++;
        totalPauseTime += pauseTime;
        maxPauseTime = Math.max(maxPauseTime, pauseTime);
        pauseTimeList.add(pauseTime);
        
        //【AOSP 17 新增】分代 GC 统计
        switch (gcInfo.gc_type) {
            case GC_TYPE_YOUNG: youngGcCount++; break;
            case GC_TYPE_OLD: oldGcCount++; break;
            case GC_TYPE_STICKY: stickyGcCount++; break;
        }
        totalPromotedObjects += gcInfo.promoted_objects;
        
        //【AOSP 17 新增】软阈值统计
        if (gcInfo.distance_to_soft_threshold > -5) {
            softThresholdReached++;
        }
        if (gcInfo.distance_to_hard_threshold > 0) {
            hardThresholdReached++;
        }
        
        // 告警检测
        if (pauseTime > 100) {
            alertManager.alert("gc.pause.long", pauseTime);
        }
        //【AOSP 17 新增】软阈值告警
        if (gcInfo.distance_to_soft_threshold > 0) {
            alertManager.alert("gc.soft.threshold.exceeded",
                gcInfo.distance_to_soft_threshold);
        }
    }
    
    //【AOSP 17 新增】ObjectFree 回调
    public void onObjectFree(String className) {
        objectFreeCount++;
        freedClassNames.add(className);
    }
    
    // 周期上报（每分钟）
    public void report() {
        if (gcCount > 0) {
            Map<String, Object> metrics = new HashMap<>();
            metrics.put("gc.count", gcCount);
            metrics.put("gc.pause.avg", totalPauseTime / gcCount);
            metrics.put("gc.pause.max", maxPauseTime);
            metrics.put("gc.pause.p95", percentile(95));
            metrics.put("gc.pause.p99", percentile(99));
            //【AOSP 17 新增】
            metrics.put("gc.young.count", youngGcCount);
            metrics.put("gc.old.count", oldGcCount);
            metrics.put("gc.sticky.count", stickyGcCount);
            metrics.put("gc.promoted", totalPromotedObjects);
            metrics.put("gc.soft.threshold.reached", softThresholdReached);
            metrics.put("gc.hard.threshold.reached", hardThresholdReached);
            metrics.put("object.free.count", objectFreeCount);
            
            reporter.report("gc.metrics", metrics);
            
            // 重置
            reset();
        }
    }
    
    private void reset() {
        gcCount = 0;
        totalPauseTime = 0;
        maxPauseTime = 0;
        pauseTimeList.clear();
        //【AOSP 17】
        youngGcCount = 0;
        oldGcCount = 0;
        stickyGcCount = 0;
        totalPromotedObjects = 0;
        softThresholdReached = 0;
        hardThresholdReached = 0;
        objectFreeCount = 0;
        freedClassNames.clear();
    }
    
    // 分位数计算
    private long percentile(int p) {
        if (pauseTimeList.isEmpty()) return 0;
        Collections.sort(pauseTimeList);
        int index = (int) Math.ceil(p / 100.0 * pauseTimeList.size()) - 1;
        return pauseTimeList.get(Math.max(0, index));
    }
}
```

### 9.10.9 JVMTI 原生库实现（AOSP 17 增强）

```cpp
// jvmti-gc-monitor.cpp（AOSP 17 增强版）
#include <jni.h>
#include <jvmti.h>
#include <android/log.h>

static jclass gcCollectorClass;
static jmethodID onGcStartMethod;
static jmethodID onGcFinishMethod;  // 接收 jvmtiGarbageCollectionInfo
static jmethodID onObjectFreeMethod;  //【AOSP 17 新增】
static jlong g_gc_start_time = 0;

//【AOSP 17 增强】GC 开始
void JNICALL OnGCStart(jvmtiEnv* env) {
    g_gc_start_time = currentTimeMillis();
}

//【AOSP 17 增强】GC 结束（含扩展元数据）
void JNICALL OnGCFinish(jvmtiEnv* env) {
    long pause_time = currentTimeMillis() - g_gc_start_time;
    
    JNIEnv* jni;
    env->GetJavaEnv(reinterpret_cast<void**>(&jni));
    
    //【AOSP 17 新增】获取 GC 扩展元数据
    jvmtiGarbageCollectionInfo gc_info = {0};
    env->GetGarbageCollectionExtendedInfo(&gc_info);
    
    // 构造 Java 对象传给 onGcFinish（需要 JNI 包装）
    // 简化：直接传递 pause_time 和 gc_info 关键字段
    jni->CallVoidMethod(currentThread, onGcFinishMethod, 
                        pause_time,
                        gc_info.gc_type,
                        gc_info.gc_cause,
                        gc_info.heap_alloc,
                        gc_info.distance_to_soft_threshold,
                        gc_info.distance_to_hard_threshold,
                        gc_info.promoted_objects);
}

//【AOSP 17 新增】对象释放
void JNICALL OnObjectFree(jvmtiEnv* env, jlong tag) {
    JNIEnv* jni;
    env->GetJavaEnv(reinterpret_cast<void**>(&jni));
    
    // 通过 tag 获取类名（需要 SetTag 时记录）
    jclass classClass = jni->FindClass("java/lang/Class");
    jni->CallVoidMethod(currentThread, onObjectFreeMethod, jstringFromTag(tag));
}

//【AOSP 17 增强】JNI 入口
JNIEXPORT void JNICALL
Java_com_example_apm_JvmtiGcMonitor_enableGcEvents(
    JNIEnv* env, jclass clazz, jobject collector) {
    
    jvmtiEnv* jvmti = nullptr;
    env->GetEnv(reinterpret_cast<void**>(&jvmti), JVMTI_VERSION_1_2);
    
    // 设置回调
    jvmtiEventCallbacks callbacks = {0};
    callbacks.GarbageCollectionStart = OnGCStart;
    callbacks.GarbageCollectionFinish = OnGCFinish;
    callbacks.ObjectFree = OnObjectFree;  //【AOSP 17 新增】
    jvmti->SetEventCallbacks(&callbacks, sizeof(callbacks));
    
    // 启用事件
    jvmti->SetEventNotificationMode(JVMTI_EVENT_GARBAGE_COLLECTION_START, JVMTI_ENABLED);
    jvmti->SetEventNotificationMode(JVMTI_EVENT_GARBAGE_COLLECTION_FINISH, JVMTI_ENABLED);
    jvmti->SetEventNotificationMode(JVMTI_EVENT_OBJECT_FREE, JVMTI_ENABLED);  //【AOSP 17 新增】
}
```

```java
//【AOSP 17 新增】ART 内部状态采集
public class ArtInternalStateCollector {
    private final Reporter reporter;
    private final Handler handler;
    private final Runnable collectRunnable;
    
    private static final long COLLECT_INTERVAL = 60_000;  // 1 分钟
    
    public ArtInternalStateCollector(Reporter reporter) {
        this.reporter = reporter;
        this.handler = new Handler(Looper.getMainLooper());
        this.collectRunnable = new Runnable() {
            @Override
```

```java
            @Override
            public void run() {
```

```java
        try {
            //【AOSP 17 + Linux 6.18】通过 smaps 读取 sheaves slab
            Process p = Runtime.getRuntime().exec(new String[]{
                "sh", "-c", "cat /proc/self/smaps"
            });

                new InputStreamReader(p.getInputStream()));
            String line;
            long heapPss = 0;
            long sheavesPss = 0;
            boolean inHeap = false;
            boolean inSheaves = false;
            
            while ((line = reader.readLine()) != null) {
                if (line.contains("[heap]")) {
                    inHeap = true;
                    inSheaves = false;
                    continue;
                }
                if (line.contains("[sheaves]")) {
                    inHeap = false;
                    inSheaves = true;
                    continue;
                }
                if (inHeap && line.startsWith("Pss:")) {
                    heapPss += parsePss(line);
                    inHeap = false;
                }
                if (inSheaves && line.startsWith("Pss:")) {
                    sheavesPss += parsePss(line);
                    inSheaves = false;
                }
            }
            
            Map<String, Object> metrics = new HashMap<>();
            metrics.put("native.sheaves.pss", sheavesPss);
            metrics.put("native.heap.pss", heapPss);
            metrics.put("native.sheaves.ratio", 
                heapPss + sheavesPss > 0 ? (double) sheavesPss / (heapPss + sheavesPss) : 0);
            
            reporter.report("native.sheaves", metrics);
        } catch (IOException e) {
            Log.e(TAG, "collectSheaves failed", e);
        }
    }
```

### 9.10.13 Reporter 实现（Linux 6.18 增强）

```java
public class Reporter {
    private final String serverUrl;
    private final String appId;
    private final ExecutorService executor;
    private final Queue<Metric> queue = new LinkedBlockingQueue<>();
    
    public Reporter(String serverUrl, String appId) {
        this.serverUrl = serverUrl;
        this.appId = appId;
        this.executor = Executors.newSingleThreadExecutor();
    }
    
    public void report(String type, Map<String, Object> metrics) {
        Metric metric = new Metric(type, appId, System.currentTimeMillis(), metrics);
        queue.offer(metric);
        flushIfNeeded();
    }
    
    // 异步上报
    private void flushIfNeeded() {
        if (queue.size() >= 10 || hasOldestMetric()) {
            executor.execute(this::flush);
        }
    }
    
    private void flush() {
        List<Metric> batch = new ArrayList<>();
        Metric m;
        while ((m = queue.poll()) != null) {
            batch.add(m);
        }
        
        if (!batch.isEmpty()) {
            try {
                //【Linux 6.18 增强】io_uring 异步 I/O 让上报更快
                OkHttpClient client = new OkHttpClient.Builder()
                    .connectTimeout(5, TimeUnit.SECONDS)
                    .writeTimeout(5, TimeUnit.SECONDS)
                    .readTimeout(10, TimeUnit.SECONDS)
                    .build();
                
                RequestBody body = RequestBody.create(
                    MediaType.parse("application/json"),
                    toJson(batch)
                );
                
                Request request = new Request.Builder()
                    .url(serverUrl + "/api/metrics")
                    .post(body)
                    .build();
                
                client.newCall(request).execute();
            } catch (IOException e) {
                // 上报失败，加入下次上报
                for (Metric failed : batch) {
                    queue.offer(failed);
                }
            }
        }
    }
    
    private boolean hasOldestMetric() {
        Metric head = queue.peek();
        return head != null && 
            System.currentTimeMillis() - head.timestamp > 30_000;
    }
}
```

### 9.10.14 AlertManager 实现（AOSP 17 增强）

```java
public class AlertManager {
    private final Reporter reporter;
    private final List<AlertRule> rules;
    
    // 告警去重（防止抖动）
    private final Map<String, Long> lastAlertTime = new HashMap<>();
    private static final long ALERT_INTERVAL_MS = 60_000;  // 同类告警 1 分钟内只发一次
    
    public AlertManager(Reporter reporter, List<AlertRule> rules) {
        this.reporter = reporter;
        this.rules = rules;
    }
    
    public void alert(String type, Object value) {
        for (AlertRule rule : rules) {
            if (rule.matches(type, value)) {
                // 去重
                String key = type + ":" + rule.getLevel();
                Long lastTime = lastAlertTime.get(key);
                long now = System.currentTimeMillis();
                
                if (lastTime == null || now - lastTime > ALERT_INTERVAL_MS) {
                    // 发送告警
                    Map<String, Object> alertData = new HashMap<>();
                    alertData.put("type", type);
                    alertData.put("level", rule.getLevel());
                    alertData.put("value", value);
                    alertData.put("timestamp", now);
                    alertData.put("stackTrace", getStackTrace());
                    
                    reporter.report("alert", alertData);
                    lastAlertTime.put(key, now);
                }
            }
        }
    }
    
    private String getStackTrace() {
        StackTraceElement[] stack = Thread.currentThread().getStackTrace();
        StringBuilder sb = new StringBuilder();
        for (StackTraceElement element : stack) {
            sb.append(element.toString()).append("\n");
        }
        return sb.toString();
    }
}
```

### 9.10.15 AlertRule 定义（AOSP 17 增强）

```java
public class AlertRule {
    private final String type;          // gc.pause.long / soft.threshold / jni.refs
    private final String comparator;    // > / < / ==
    private final double threshold;
    private final String level;         // warning / alert / critical
    
    public boolean matches(String alertType, Object value) {
        if (!this.type.equals(alertType)) return false;
        
        double numValue = ((Number) value).doubleValue();
        switch (comparator) {
            case ">": return numValue > threshold;
            case "<": return numValue < threshold;
            case "==": return numValue == threshold;
            default: return false;
        }
    }
    
    public String getLevel() { return level; }
}
```

5. **告警规则**（AOSP 17 增强）：阈值 + 去重 + 严重级别 + 软阈值 + JNI 泄漏
6. **最佳实践**：采样上报 + 异步批量（Linux 6.18 io_uring）+ 失败重试

→ **理解自建 APM 集成 + AOSP 17 增强 + Linux 6.18 优化，就掌握了"线上 GC 监控 + ART 17 适配"的生产级方案**。

## 十三、总结（架构师视角的 5 条 Takeaway）

1. **自建 APM 是生产环境 GC 监控的"终极方案"**——App 端 SDK + 服务端 + Grafana。**AOSP 17 必选**：GC 扩展元数据 + ObjectFree + ART 内部状态。详见 §2 + [06-JVMTI监控GC](06-JVMTI监控GC.md)（重写为 v2 升级版）。

2. **AOSP 17 JVMTI GC 扩展元数据是 APM 升级的核心**——直接拿到 GC 类型、原因、软阈值距离、晋升数。**不用估算、不用拉取**，GC 回调中即可拿到。详见 §4 + [06-JVMTI监控GC](06-JVMTI监控GC.md) §6.2。

3. **AOSP 17 ObjectFree 集成让泄漏检测更精准**——每个对象释放时触发，**未释放的就是泄漏候选**。**采样上报是关键**（高频事件需采样）。详见 §4.7 + [03-LeakCanary原理](03-LeakCanary原理.md)（重写为 v2 升级版）。

4. **AOSP 17 + Linux 6.18 sheaves slab 监控让 Native 堆分析更精准**——通过 smaps 读取 sheaves 段，**不要把 sheaves 优化误判为泄漏修复**。详见 §5.6 + [02-procrank-smaps](02-procrank-smaps.md) §6.1。

5. **AOSP 17 告警规则必须升级**——软阈值告警（30%/60 次/100 次）+ JNI 泄漏告警（1000/5000/10000）+ 晋升速率告警。**关键预警全部上报**，普通事件采样 1/10。详见 §7.6 + [07-监控指标体系](07-监控指标体系.md)（重写为 v2 升级版）。
