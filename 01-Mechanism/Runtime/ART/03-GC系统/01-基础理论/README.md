# 01-基础理论：可达性、三色标记、写/读屏障、记忆集、Reference 体系

> **本篇是 GC 系列的"理论根基"** —— 不懂这一篇，后面的 8 篇源码都是空中楼阁。
>
> **本篇不讲任何具体 GC 算法**（CMS / CC / GenCC 都在后面）。只讲 **所有 ART GC 算法共享的底层机制**。

---

## 一句话定位

**ART 的 GC 是一个"三色标记-复制"算法家族**。无论 CMS（标记-清除）还是 CC / GenCC（标记-复制），都共享同一套理论根基：
- **可达性分析**——怎么决定"对象死还是活"
- **三色不变式**——并发标记正确性的数学表达
- **写屏障 / 读屏障**——并发场景下维护不变式的钩子
- **记忆集 / 卡表**——分代 GC 的工程优化
- **Reference 体系**——Java 引用语义的对象回收时机

这 5 个机制是 **所有 GC 算法的通用语言**。
