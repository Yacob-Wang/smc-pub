# 03-CMS-GC：标记-清除的并发艺术

> **本篇是 GC 系列的"算法篇 1"** —— 把 ART Android 5.0-7.0 默认的 CMS（Concurrent Mark Sweep）GC 算法讲透。
>
> **理解本篇，就理解了"为什么 Android 8.0 要换 CC GC"** —— CMS 是 CC 的"反面教材"。

---

## 一句话定位

**CMS = Concurrent Mark Sweep = 并发标记-清除**。ART Android 5.0-7.0 的默认 GC 算法。
核心思想：**在 STW 暂停极短的前提下，并发完成大部分标记和清除工作**。

但 CMS 有 **三大硬伤**：
1. **STW 时间不可控**（Remark 可能飙到 50ms+）
2. **内存碎片化**（不压缩 + Run-of-Slots 分桶）
3. **写屏障成本**（Incremental Update 每次指针赋值都要拦截）

正是这三大硬伤，催生了 ART 8.0 的 **CC GC**（详见 04 篇）。
