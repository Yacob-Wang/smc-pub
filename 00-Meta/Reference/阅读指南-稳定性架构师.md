## 1. 网站导航 ↔ 阅读顺序

顶栏 Tab 顺序（与 `content_policy.py` 一致）：

```
00-Meta → 01-Mechanism → 02-Symptom → 03-Forensics → 04-Tool → 05-Governance → 06-Case → 06-Foundation
```

**推荐工作流闭环**：

```
06-Foundation（编译/刷机）→ 01-Mechanism（机制/源码）
  → 02-Symptom（症状归类）→ 03-Forensics（取证剧本）→ 04-Tool（工具实操）
  → 05-Governance（度量/APM/门禁）→ 06-Case（实战复盘）→ 反哺治理
```

**应急跳转**：站点首页「按问题进入」—— Native Crash / ANR / OOM / Watchdog / 启动 / 度量门禁 等一键入口。

**侧栏策略**：模块页 = 系列卡片；进入某系列后侧栏只显示该系列单篇，避免 500+ 篇堆在一起。

| 3 | ANR 三件套 | S01 + [ANR-Detection](../../04-Tool/ANR-Detection/) 3 篇 + [Input/06](../../01-Mechanism/Framework/Input/) + [Handler/06](../../01-Mechanism/App/Handler-MessageQueue-Looper/Handler_MessageQueue_Looper/) |

**5 个交叉点**（面试/排查必背）：

1. 主线程卡死 → ANR → SWT
2. Binder 饿死 → ANR
3. GC 频繁 → 卡顿 → ANR
4. 内存泄漏 → OOM → 进程死
5. Native 崩溃 → tombstone 全链路

## 6. 阅读节奏与反例

**3-2-1 节奏**（总路线图 §12）：

- **3**：每天 3 篇机制文（深度读，含源码）
- **2**：每天 2 篇工具/案例文（边读边练）
- **1**：每天 1 次实操（dumpsys / perfetto）

**反例**：

- ❌ 跨阶段跳读（机制未立就读症状）
- ❌ 只收藏不练
- ❌ 孤立学习（不跟 oncall 群）

**先查质量**：[02-Symptom/README-质量评估.md](../../02-Symptom/README-质量评估.md) 标记 🟠 的篇章谨慎读。
